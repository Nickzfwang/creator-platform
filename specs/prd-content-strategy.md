# AI 內容策略引擎 — 產品需求文檔 (PRD)

> Status: draft | Owner: Nick | Priority: P0
> Created: 2026-03-25 | Target: TBD

## 1. 背景與目標

### 問題陳述

創作者最大的痛點不是「不會剪」而是「不知道拍什麼」。選題決策高度依賴直覺，缺乏數據支撐，導致：
- 產出不穩定：有靈感時密集更新，沒靈感時停更數週
- 流量波動大：爆款靠運氣，無法系統化複製成功
- 錯過趨勢窗口：等到話題退燒才跟上，失去先發優勢
- 缺乏長期規劃：沒有內容日曆，無法維持穩定的觀眾預期

**核心痛點：創作者需要一個數據驅動的「內容顧問」，告訴他下一支該拍什麼、什麼時候發。**

### 目標

- 整合頻道歷史表現 + 外部趨勢 + 競品分析，AI 推薦最有潛力的影片主題
- 自動生成內容日曆，創作者只需審核和調整
- 追蹤「AI 建議 → 採用 → 實際表現」的完整閉環，持續優化推薦品質

### 成功指標

| 指標 | 目前值 | 目標值 |
|------|--------|--------|
| 創作者選題決策時間 | 數小時~數天 | < 10 分鐘（從 AI 建議中挑選） |
| 內容產出穩定度（每週更新率） | 不規律 | 每週穩定 2-3 支（依日曆規劃） |
| AI 建議採用率 | N/A | > 40%（建議被採納或改編） |
| 採用 AI 建議的影片平均表現 | N/A | 優於頻道平均觀看數 20% |

## 2. 用戶故事與驗收條件

### Story 1: AI 主題推薦

**作為** 內容創作者，**我想要** AI 根據我的頻道數據和外部趨勢推薦影片主題，**以便** 我不再靠直覺選題，而是有數據支撐的決策依據。

**驗收條件 (Acceptance Criteria):**
- [ ] AC1: Given 創作者有歷史影片數據, When 點擊「AI 推薦主題」或系統每週自動觸發, Then 生成 5-10 個主題建議
- [ ] AC2: Given 主題建議已生成, When 創作者查看建議列表, Then 每個建議包含：標題、簡述、推薦理由（引用具體數據來源）、預估表現分數（1-10）、信心指標（高/中/低）
- [ ] AC3: Given 創作者查看建議, When 選擇推薦偏好（趨勢導向/數據導向/競品導向）, Then AI 根據偏好重新排序或生成新建議
- [ ] AC4: Given 創作者是新用戶（無歷史數據）, When 觸發 AI 推薦, Then fallback 到趨勢導向推薦 + 要求用戶選擇 niche 類別

### Story 2: 內容日曆管理

**作為** 內容創作者，**我想要** 在日曆視圖中管理我的內容計畫，**以便** 我能維持穩定的更新節奏並提前規劃。

**驗收條件 (Acceptance Criteria):**
- [ ] AC1: Given AI 已生成主題建議, When 創作者點擊「排入日曆」, Then 主題被加入內容日曆，包含建議的發佈日期和時間
- [ ] AC2: Given 內容日曆有項目, When 創作者查看日曆頁面, Then 顯示月/週視圖，每個項目顯示主題、狀態（planned/in-production/published）、目標平台
- [ ] AC3: Given 內容日曆有項目, When 創作者拖拉調整日期, Then 項目日期即時更新
- [ ] AC4: Given 創作者確認某個日曆項目, When 點擊「開始製作」, Then 狀態轉為 in-production，可關聯到實際影片上傳

### Story 3: 每週自動生成計畫

**作為** 內容創作者，**我想要** 每週一自動收到 AI 生成的本週內容計畫，**以便** 我打開平台就知道本週該做什麼。

**驗收條件 (Acceptance Criteria):**
- [ ] AC1: Given 創作者已設定 niche 和偏好, When 每週一早上 9:00（依用戶時區）, Then 系統自動生成本週 3-5 個主題建議並排入日曆草稿
- [ ] AC2: Given 自動生成完成, When 創作者登入, Then 在儀表板看到「本週內容計畫已就緒」的提示卡片
- [ ] AC3: Given 創作者查看自動生成的計畫, When 不滿意某個建議, Then 可刪除或點擊「換一個」替換單個建議

### Story 4: 競品追蹤

**作為** 內容創作者，**我想要** 追蹤同領域創作者的內容策略，**以便** 我可以學習成功模式並找到差異化機會。

**驗收條件 (Acceptance Criteria):**
- [ ] AC1: Given 創作者在設定頁, When 輸入競品 YouTube 頻道 URL, Then 系統抓取該頻道的公開影片列表（最近 50 支）
- [ ] AC2: Given 已追蹤競品頻道, When 創作者查看競品面板, Then 顯示競品近期影片主題、觀看數、發佈頻率
- [ ] AC3: Given 有競品數據, When AI 生成主題建議, Then 建議中包含「競品機會」類型：競品做了但你沒做的熱門主題、競品還沒做但趨勢看好的主題
- [ ] AC4: Given 競品頻道已追蹤, When 競品發佈新影片, Then 每日自動同步更新（BullMQ cron）

### Story 5: 策略回顧與優化

**作為** 內容創作者，**我想要** 回顧 AI 建議的實際成效，**以便** AI 能持續學習並給出更好的建議。

**驗收條件 (Acceptance Criteria):**
- [ ] AC1: Given 有被採用的 AI 建議且對應影片已發佈, When 創作者查看「策略回顧」頁面, Then 顯示：建議主題 vs 實際標題、預估分數 vs 實際觀看數/互動率
- [ ] AC2: Given 有歷史回顧數據, When AI 分析成效, Then 產出洞察報告：哪類建議表現最好、哪些數據信號最準、建議下月策略調整方向
- [ ] AC3: Given 創作者查看回顧報告, When 按月/季篩選, Then 可看到不同時期的建議採用率和成效趨勢

## 3. 功能規格

### 3.1 AI 主題推薦引擎（後端核心）

- **描述**: 整合多個數據源，透過 GPT-4o 生成結構化主題建議
- **數據來源**:
  - **頻道歷史**: analytics 模組 → getTopContent()、getOverview()
  - **外部趨勢**: trend-radar 模組 → getTrends()
  - **競品數據**: 新增 CompetitorService → YouTube Data API v3 公開影片列表
- **AI 模型**: GPT-4o（需要綜合推理多個數據源 + 策略思考）
- **輸出結構**:
```typescript
interface TopicSuggestion {
  id: string;
  title: string;                    // 建議影片標題
  description: string;              // 主題簡述（2-3 句）
  reasoning: string;                // 推薦理由（引用具體數據）
  dataSource: 'HISTORY' | 'TREND' | 'COMPETITOR' | 'MIXED';
  performanceScore: number;         // 1-10 預估表現
  confidenceLevel: 'HIGH' | 'MEDIUM' | 'LOW';
  confidenceReason: string;         // 信心指標的依據
  suggestedDate: string;            // 建議發佈日期
  suggestedPlatforms: string[];     // 建議發佈平台
  tags: string[];                   // 相關標籤
  relatedTrends?: string[];         // 相關趨勢話題
  competitorRef?: string;           // 競品參考（影片 URL）
}
```

- **業務規則**:
  - 新用戶（無歷史數據）→ 只用趨勢 + 要求選擇 niche
  - 建議不重複近 30 天已建議過的主題（embedding 相似度檢查）
  - 每次生成 5-10 個建議，按 performanceScore 降序排列

### 3.2 內容日曆（後端）

- **描述**: 管理內容規劃的時間線，連結 AI 建議與實際產出
- **資料模型**: 新增 `ContentCalendar` 資料表
- **狀態機**:
```
SUGGESTED → PLANNED → IN_PRODUCTION → PUBLISHED → MEASURED
    ↓          ↓
 DISMISSED   SKIPPED
```
- **業務規則**:
  - 每個日曆項目可關聯一個 TopicSuggestion（來源追蹤）
  - PUBLISHED 狀態可關聯實際影片 ID（用於成效回顧）
  - 支援手動新增日曆項目（非 AI 建議）
  - 與 post-scheduler 串接：日曆項目確認後可一鍵建立排程 Post

### 3.3 競品追蹤（後端）

- **描述**: 追蹤並分析競品頻道的內容策略
- **資料模型**: 新增 `Competitor` + `CompetitorVideo` 資料表
- **數據來源**: YouTube Data API v3（公開頻道影片列表）
- **同步策略**: BullMQ cron job，每日同步一次
- **業務規則**:
  - 每位創作者最多追蹤 10 個競品頻道（Free: 3, Starter: 5, Pro: 10, Business: 10）
  - 只抓取公開資料（標題、觀看數、讚數、發佈日期、縮圖）
  - 不儲存影片內容本身

### 3.4 策略回顧分析（後端）

- **描述**: 追蹤 AI 建議的完整生命週期，分析成效
- **AI 模型**: GPT-4o-mini（彙總分析，非複雜推理）
- **分析維度**:
  - 建議採用率（adopted / total suggested）
  - 預估分數 vs 實際表現的偏差
  - 各數據源（歷史/趨勢/競品）的建議成效比較
  - 月度/季度趨勢
- **輸出**: 結構化洞察報告 + 策略調整建議

### 3.5 前端頁面

- **新增頁面**: `/strategy` — 內容策略主頁面
- **子功能**:
  - AI 推薦面板：建議列表 + 篩選（偏好/數據來源） + 操作（排入日曆/忽略/換一個）
  - 內容日曆：月/週視圖 + 拖拉 + 狀態標示
  - 競品追蹤：頻道列表 + 近期影片 + 趨勢分析
  - 策略回顧：成效圖表 + AI 洞察報告

## 5. 範圍定義

### In Scope（本次實作）

- AI 主題推薦 API（整合 analytics + trend-radar + 競品數據）
- 推薦偏好設定（趨勢/數據/競品/綜合）
- 內容日曆 CRUD + 日曆視圖 UI
- 日曆項目狀態機（SUGGESTED → PLANNED → IN_PRODUCTION → PUBLISHED → MEASURED）
- 與 post-scheduler 串接（日曆 → 排程 Post）
- 競品頻道追蹤（YouTube，手動新增 URL）
- 競品影片資料每日同步（YouTube Data API v3）
- 每週自動生成內容計畫（BullMQ cron）
- 策略回顧分析（建議 vs 實際成效比對 + AI 洞察報告）
- 新用戶 fallback（趨勢導向 + niche 選擇）
- 前端 /strategy 頁面（推薦 + 日曆 + 競品 + 回顧 4 個 Tab）
- 儀表板「本週計畫就緒」提示卡片

### Out of Scope（未來考慮）

- AI 自動推薦競品頻道（需搜尋 API）
- 非 YouTube 平台的競品追蹤（IG/TikTok 公開 API 限制多）
- SEO 關鍵字整合（Google Trends API / Search Console）
- A/B 標題測試
- 團隊協作（多人編輯日曆）
- 推播通知（推薦完成通知）
- 自動將日曆項目轉為影片 shot list / 腳本

## 6. 非功能需求

- **效能**: AI 主題推薦生成 < 30 秒（並行查詢多個數據源 + 單次 GPT-4o 呼叫）
- **成本**: 單次推薦生成（5-10 個主題）的 AI 成本 < $0.15（GPT-4o 單次呼叫）
- **可靠性**: 每週自動生成失敗時重試 3 次，間隔指數退避；失敗不影響手動觸發
- **安全**: YouTube Data API key 不暴露至前端；競品數據僅限公開資訊
- **多租戶**: 所有資料模型遵循 tenantId 隔離
- **API 配額**: YouTube Data API v3 每日 10,000 units quota，需追蹤用量避免超額

## 7. 風險與未決事項

| 項目 | 影響 | 對策 |
|------|------|------|
| AI 推薦品質不穩定 | 創作者不信任建議，降低採用率 | 每個建議附推薦理由（引用具體數據），讓用戶可判斷；透過回顧閉環持續優化 prompt |
| 新用戶無歷史數據 | 推薦缺乏個人化 | Fallback 到趨勢導向推薦 + niche 選擇 + 逐步累積數據 |
| YouTube Data API 配額限制 | 競品同步頻率受限 | 每日同步一次 + 追蹤 quota 用量 + 超額時降級為手動觸發 |
| 「預估表現」與實際的偏差 | 影響用戶信任 | 明確標示為「參考指標」；顯示信心等級 + 理由；回顧分析持續校準 |
| 建議→採用→成效的追蹤鏈斷裂 | 回顧分析不準確 | 設計流暢的「關聯影片」操作；PUBLISHED 時主動提示關聯 |
| 競品數據只限 YouTube | 限制多平台創作者的使用 | MVP 先做 YouTube（API 最開放），其他平台列入 Out of Scope |
